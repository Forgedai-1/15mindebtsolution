Thank you for your interest in contributing to 15min.debtsolution!

Guidelines
- Fork the repository and create your branch from the default branch (usually `main`).
- Run tests and linters before submitting a PR.
- Keep changes small and focused; include tests for new behavior.
- Write clear commit messages and an informative PR description.

How to contribute
1. Fork the repo and create a feature branch: `git checkout -b feature/your-feature`
2. Make your changes and add tests if applicable.
3. Commit with a clear message: `git commit -m "Add feature: ..."`
4. Push to your fork and open a Pull Request against the repository's default branch.

Code style and tests
- Follow the project's coding conventions (to be documented in README).
- Add or update tests for any new features or bug fixes.

Reporting issues
- Open an issue with a clear title and a short reproduction, expected vs actual behavior, and any relevant logs or stack traces.

Thank you for helping improve 15min.debtsolution!
